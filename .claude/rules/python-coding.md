## コーディング規約

- PEP8に従う
- 型ヒント必須（パラメータと戻り値）
- KISSの原則に従い、できるだけシンプルなコードにする
- 可読性を優先する（一度読んだだけで理解できるコードが最高）

## import順

標準ライブラリ → サードパーティ → ローカル（各グループ内アルファベット順、`import` 先・`from` 後）

## クリーンコード

- 関数は50行以下を目標。処理が多すぎる場合は小さな関数に分割する
- 各関数とモジュールには明確な目的を1つ持たせる
- 説明的な名前を使う（`tmp`、`data`、`doCalc` などの一般的な名前は避ける）
- 類似のロジックが2箇所に存在する場合は共有関数にリファクタリングする

## コメント

- 分かりにくいロジックにのみ日本語で説明を加える
- 説明不要のコードには過剰なコメントをつけない
- コメントとdocstringは必要最小限に日本語で記述する

## UI メッセージ

- UI画面で表示するメッセージはすべて日本語にする
- `constants.py` で一元管理し、マジック文字列を使わず必ず定数を参照する
